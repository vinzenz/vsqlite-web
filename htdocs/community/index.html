Where to get help

  • Issues & bugs: use the GitHub issue tracker for reproducible problems, crash reports, or feature requests.
  • Questions: start a thread in GitHub Discussions for usage questions, design topics, or showcase posts.
  • Security disclosures: please use the GitHub Security Advisory form so maintainers can coordinate a private fix before disclosure.

Contributing

  • Read the contribution guide for coding style, CMake options, and testing expectations.
  • Open PRs against master. Include a short summary, testing notes (cmake --build build && ctest), and mention any API changes.
  • For larger features, draft a proposal in Discussions first so maintainers can help shape the scope.

Community expectations

  • Be respectful, welcoming, and patient. We value constructive feedback.
  • Keep discussions on topic; move off-topic chat to community threads.
  • Signal when you stop working on an issue so others can pick it up.

Stay updated

  • Watch GitHub releases for tagged versions and changelogs.
  • Follow maintainers on Mastodon/GitHub to catch release highlights.
  • Bookmark the homepage for build instructions and feature news.
